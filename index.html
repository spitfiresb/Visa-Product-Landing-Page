Extending Visa Token Service for AI agents

The Agent Identity Token (AIT) Framework extends Visa Token Service to create persistent, cryptographically-secured agent identifiers that travel with transactions throughout the entire payment lifecycle—from authorization through settlement to dispute resolution.

While Visa's Trusted Agent Protocol verifies agent legitimacy at the point of transaction, that verification currently exists only at the session level. AIT creates a programmable, domain-controlled, tokenized, and authenticated agent identity layer that enables issuers, merchants, and Visa to track which specific agent acted on a consumer's behalf across the full transaction journey.

Enabling trust across the ecosystem

Build, protect, and authenticate agent-initiated transactions.

Merchants

AIT enables merchants to identify, track, and optimize for agent-initiated transactions with full visibility into which AI agents are acting on behalf of consumers.

Potential benefits:

  • Distinguish agent transactions from human transactions for tailored checkout experiences
  • Access analytics on agent purchasing patterns and product recommendations
  • Reduce fraud exposure with cryptographically verified agent identities
  • Streamline dispute resolution with immutable agent audit trails

Issuers

Financial institutions can leverage AIT to validate agent permissions against cardholder-defined constraints in real-time during authorization.

Potential benefits:

  • Validate agent permission scope matches transaction parameters
  • Enforce spending limits, merchant categories, and time windows set by cardholders
  • Enhanced fraud detection through agent behavior analytics
  • Clear audit trail for regulatory compliance and dispute resolution

Agent Issuers

AI platforms like Anthropic, Google, and OpenAI can register their agents with Visa to receive unique AITs that establish trust across the payment network.

Potential benefits:

  • Establish verified identity for agents across all Visa merchants
  • Access aggregated fraud intelligence and high-risk category data
  • Receive Visa transaction data to improve agent decision-making models
  • Build consumer trust through Visa-verified agent credentials

Consumers

Cardholders maintain full control over agent permissions with granular controls and instant revocation capabilities.

Potential benefits:

  • Set specific spending limits, merchant categories, and time windows for each agent
  • View complete history of agent-initiated transactions
  • Instantly revoke agent permissions at any time
  • Clear visibility into which agent acted during disputes

AIT Data Scheme

Structured metadata encoded in a cryptographic hash accompanying each transaction.

Agent ID

Unique identifier for the specific agent instance

Developer/Platform ID

Identifier of the agent issuer (Anthropic, Google, etc.)

Merchant Category

Authorized MCC codes the agent can transact with

Spending Limits

Per-transaction and cumulative spending caps

Time Window

Valid transaction period for agent authorization

Consumer Context

Authorization scope and permission hash

Validation points

Where and when AIT gets verified in the transaction flow.

At Checkout

Merchant validates agent is authorized to transact and meets merchant-specific requirements

During Authorization

Issuer validates permission scope matches transaction amount, merchant category, and time window

During Disputes

All parties can validate agent identity and authorization context for chargeback resolution
